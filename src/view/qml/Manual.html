Noteahead User Manual

Noteahead is a "simple" MIDI tracker and sequencer for Linux focusing on ease of use. It is designed for fast, keyboard-driven sequencing of external synths and drum machines, with clear handling of patch changes, automation, and CCs — no cryptic hex values required.

The Toolbar

The toolbar at the top of the window provides essential controls for song structure, pattern editing, and global settings.

Song Section

  • Player Controls: Play, Previous, Stop, and Loop buttons.
  • POS: The current position in the Song Play Order (0-indexed).
  • Skipped Patterns: Right-click on a position button to toggle its "skipped" state. Skipped positions are highlighted in red and will be automatically bypassed during playback. This also immediately updates the total song duration. Note that skipping can only be toggled when the song is stopped.
  • PAT: The Pattern Index assigned to the current song position. Changing this allows you to reuse patterns throughout your song.
  • LEN: The total length of the song (number of positions in the play order).
  • [+] / [-]: Insert or remove a position in the song play order.

Pattern Section

  • Pattern Name: Click the orange text to rename the active pattern.
  • PAT: The Active Pattern Index being edited. You can switch between patterns here to work on different parts of your project.
  • LEN: The length of the current pattern in lines (e.g., 64). Each pattern can have a different length.

Edit Section

  • STEP: The edit step. When you enter a note, the cursor automatically moves down by this number of lines. Set to 0 to stay on the same line.
  • VEL: The default velocity (0-127) assigned to new notes you input.
  • OCT: The base octave (0-8) for note entry.

Tempo Section

  • BPM: Beats Per Minute. Sets the overall playback speed.
  • LBP: Lines Per Beat. Defines how many tracker lines make up one musical beat (usually 4 for a standard 4/4 signature).

Editor Columns

Each Note Column consists of three main data fields per line:

C-4 100 00

  1. Note (e.g., C-4): The musical pitch and octave. "OFF" indicates a Note Off event, which stops the playing note.
  2. Velocity (000-127): How hard the note is struck. 127 is maximum, 0 is silent.
  3. Delay (00-XX): Sub-line timing delay. This allows you to "humanize" your sequence by starting a note slightly after the line's exact beat. The maximum value depends on your Ticks Per Line setting.

Core Concepts

Tracks and Note Columns

Each track in Noteahead represents one MIDI instrument. While you can change patches on-the-fly, the track-based approach keeps your sequence organized.

  • Track Settings: Set the MIDI port, channel, bank, patch, volume, pan, and cutoff.
  • Drum track: When enabled in Track Settings, the track is excluded from pattern-wide transposition commands. This is useful for tracks triggering drum kits where transposition would change the instrument instead of the pitch.
  • Note Columns: Use the [+] and [-] buttons in the track header to manage multiple note columns per track, allowing for polyphonic sequencing or complex layered parts.
  • Velocity Scaling: Velocity is calculated as the product of track scale, column scale, and individual note velocity.

Internal Timing

Noteahead uses a high-precision internal timing strategy. The song is rendered into events just before playback, and the player thread syncs to these pre-calculated timestamps to ensure jitter-free and drift-free MIDI output.

Basic Workflow

Getting Started

  1. Setup MIDI: Click the settings icon on a track header to select your MIDI output device and channel.
  2. Edit Mode: Press ESC to enter edit mode. The cursor will change color to indicate you can now input notes.
  3. Inputting Notes: Use your PC keyboard as a virtual piano (e.g., Z is C, S is C#, X is D).
  4. Playback: Press SPACE to start or stop playback.

Main Context Menu

The main context menu can be accessed by right-clicking anywhere in the editor view. It provides a comprehensive set of tools for manipulating your sequence at different scopes.

  • Line: Insert or remove specific events (like Patch Changes), set sub-line delays, and manage line-specific automations.
  • Column: Perform Cut/Copy/Paste and transposition on the current note column. You can also Add Note OFF to quickly insert a note-off event on the first line of the column, Interpolate velocity across the entire column, or manage column-wide automations.
  • Track: Manage the current track (Insert/Delete), Add Note OFF to all columns in the track (first line), transpose all columns within the track, and edit track-specific automations.
  • Pattern: Add Note OFF to all columns in the entire pattern (first line), transpose, or perform clipboard operations for the entire active pattern.
  • Delete unused patterns: Found in the Edit menu, this tool scans the play order and removes all patterns that are not assigned to any song position. This is useful for cleaning up patterns that were created during the composing process but are no longer part of the song.
  • Selection: When a range of lines/columns is selected, this menu allows you to perform actions specifically on that selection, such as adding automations or interpolating velocity.
  • Global Automation: At the bottom of the menu, you can find options to edit ALL MIDI CC or Pitch Bend automations in the project.

Advanced Features

Column Settings

Accessed by clicking the wheel icon in the column header or by right-clicking a note column and selecting Column settings, this dialog allows you to fine-tune the behavior of individual columns using three tabs:

  • Instrument: Set the Transpose value for all notes in the column. This value is added to the track-level transpose.
  • Timing: Adjust the Delay (ms) to shift all notes in the column forward or backward in time.
  • MIDI Effects: Manage Chord Automation, the Arpeggiator, and MIDI Delay for the column.

Chord Automation & Arpeggiator

Located in the MIDI Effects tab of the Column Settings, these tools allow you to generate complex harmony and movement from a single note.

  • Chord Automation: Set semi-tone offsets for up to three additional notes relative to your root note. Each chord note has its own velocity and delay settings, allowing for thick, humanized chords.
  • Arpeggiator: When enabled, the chord notes (including the root note) are played sequentially.
    • Patterns: Choose from Up, Down, Up-Down, or Random.
    • Events per beat: Determine the speed of the arpeggio relative to the BPM.

MIDI Delay

Also located in the MIDI Effects tab, this effect adds delayed echoes to all notes in the column.

  • Lines: Set the delay time between echoes in musical lines (e.g., 1.5 lines).
  • Feedback (%): Adjust the velocity of each subsequent echo relative to the previous one (0-100%).
  • Max repetitions: Set the maximum number of echoes to generate.

MIDI CC & Pitch Bend Automation

  • Add Automation: Select a range of lines, right-click, and choose "Add MIDI CC Automation" or "Add Pitch Bend Automation".
  • Interpolation & Modulation: Choose Linear for steady changes, or add Modulation (Sine Wave or Random) for organic, oscillating, or experimental movements.

MIDI Side-chain

Located in Track settings, this allows one track/column to trigger MIDI CC events on another, perfect for "pumping" effects or rhythmic modulation.

Internal Devices

Sampler

Accessed via Devices => Sampler... or Right-click => Song => Devices => Sampler..., this internal instrument allows you to trigger audio samples directly within Noteahead.

  • Pads: The sampler features 16 pads, mapped to MIDI notes 36 to 51.
  • Loading Samples: Click an empty pad to open a file dialog and select a WAV file. Once loaded, the pad will turn green.
  • Playback: Press and hold a pad to play the sample, and release to stop. In the sequencer, use the corresponding MIDI note to trigger the sample.
  • Panning: Each pad has its own base Pan setting (-100% to +100%). This can be adjusted using the slider below the waveform view.
  • Volume: Each pad has its own base Volume setting (0-100%). This can be adjusted using the slider below the pan slider.
  • Automation: The Sampler supports real-time MIDI CC 7 (Volume) and MIDI CC 10 (Pan) automation. Volume automation multiplies the pad's base volume, while pan automation is added to the pad's base pan setting.
  • Management: Right-click a loaded pad to clear its assignment.

Synchronization

Noteahead can be synchronized with external audio hosts and MIDI hardware. These options are available in Settings => MIDI => Synchronization.

  • Jack Transport Sync: When enabled, Noteahead acts as a JACK client and will follow the Play, Stop, and Rewind commands from a JACK transport master (like Carla or QjackCtl).
  • Jack BPM Sync: (Requires Jack Transport Sync) Noteahead will follow the BPM of the JACK host and synchronize its internal timeline to the JACK sample clock for drift-free, sample-accurate playback. The BPM display in the toolbar will update to reflect the host's tempo.

    • Note on Audio Recording & Playback: When Jack Transport Sync is enabled, Noteahead switches its audio recorder and player to a specialized JACK mode. This ensures stable, slaved recording and playback that is immune to system buffer size changes (PipeWire quantum changes). In this mode, Noteahead provides virtual input ports (input_1 and input_2) and output ports (output_1 and output_2) in your JACK graph. Use a patchbay like Carla or qpwgraph to route your audio sources (e.g., system:capture) to Noteahead's inputs and its outputs to your speakers or mixer.

  • MIDI Transport Sync: Allows Noteahead to be started and stopped by external MIDI controllers. Support is included for standard MIDI Start/Stop messages as well as common hardware CC mappings (e.g., Arturia Keystep).

Tools & Audio

  • Audio Recorder & Player: Enable in Settings => Audio to capture your performance directly from your system's default audio input and play back recorded files.
    • Wave View: When enabled, a waveform of the recording is shown at the bottom of the editor.
    • Playback: Use the play button next to the waveform to start or stop audio playback.
    • Navigation: Left-click on the wave view to seek both the audio playback and the main sequencer to that position. Right-click to open the folder containing the recorded file.
    • Output Device: Select your preferred playback device in the audio settings. When JACK is enabled, routing is managed via external JACK tools.
    • Persistence: The path to the last recorded file is saved with your project, allowing for seamless workflow across sessions.
  • Delay Calculator: Quickly find millisecond values for various beat divisions based on the current BPM.
  • Note Frequency Calculator: Reference tool for sound design and tuning.
